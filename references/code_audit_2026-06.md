# 全流程代码审查与最小必要优化

审查范围包括前端六个可见步骤、公共状态和通知、FastAPI 接口、文件生命周期、Mask 构建、旁白与音频、Remotion 渲染、验证脚本、启动脚本和生产文档。

## 审查原则

- 不做与当前问题无关的大规模重构。
- 优先修复用户可见错误、数据竞态、生产路径混用和缺少验证的问题。
- 保留可用的历史诊断工具，但禁止它们进入生产路径。
- 所有行为变化都增加可执行检查或浏览器验证。

## 公共界面

发现：

- 每个 Toast 自己使用 `position: fixed`，多条通知会重叠。
- 通知固定在右下角，会遮挡当前步骤的操作区域。
- `config_effectiveness.js` 在运行时覆盖 `app.js` 中的函数，形成两套实现。

优化：

- Toast 统一放入左侧步骤栏下方的固定容器，按列堆叠。
- 移动端改为底部横向安全区域。
- 将设置刷新、规则重新规划和提示词刷新逻辑合并回 `app.js`。
- 删除运行时补丁脚本和两份过期补丁说明。

## 步骤 1：导入文章

检查：

- 空文章在前端阻止提交。
- 后端保存文章后更新步骤状态。
- 修改文章会使下游结果进入重新确认状态。

本次无需额外结构调整。

## 步骤 2：分镜规划

发现：

- “保存规则”未明确说明规则只影响下次规划。
- “保存并重新规划”依赖运行时补丁动态插入。

优化：

- 将两个操作直接写入 HTML 和主脚本。
- 普通保存明确提示生效时机。
- 重新规划按钮保存后直接调用现有规划流程。

## 步骤 3：图片生成

发现：

- 页面加载没有等待图片列表请求完成，可能短暂显示空状态。
- 只要存在一张图片就显示可用的确认按钮，但后端要求全部图片存在。
- 标题固定保留 38px 高度，单行标题与图片间距过大。
- 图片操作使用内联 JavaScript 事件，维护和转义风险高。

优化：

- 等待图片列表加载完成后再结束步骤初始化。
- 所有图片齐全前禁用确认按钮，并显示缺失数量提示。
- 去掉标题固定高度，将标题和预览间距压缩到必要范围。
- AI 生成、上传和删除改为主脚本绑定事件。

## 步骤 4：Mask 标注

发现：

- 自动保存进行中点击预览时，可能读取上一次已落盘数据。
- 生产校验曾允许 blocking warning 继续通过。

优化：

- 使用 Promise 串行化 Mask 保存；预览和确认等待最新保存完成。
- 生产构建不再传递 `--allow-blocking-warnings`。
- 保留精确抠除预览和未覆盖内容诊断。

## 步骤 5：旁白与音频

检查：

- 旁白编辑、自动保存、TTS 生成和音频确认状态已串联。
- 旁白变化会清除旧音频确认。
- 每页保留尾部缓冲，防止切页截断。

现有回归检查覆盖确认状态和尾部缓冲，本次不增加额外流程。

## 步骤 6：视频合成

检查：

- 渲染前重建 Reveal 和 Remotion 运行时目录。
- 未确认音频时阻止渲染。
- MP4 有版本 sidecar，旧视频有明显标记。
- 删除视频同步删除本地 MP4 和 sidecar。
- 文件名使用 basename 和 `.mp4` 后缀校验，未发现路径穿越。

本次保留同步渲染模式；项目为单用户本地应用，引入任务队列不属于最小必要修改。

## Mask 与旧算法

发现：

- Web 生产路径已是 `manual_mask_exact_v2`，但 README、AGENTS、预检脚本和兼容入口仍描述或调用旧自动框选/分层流程。

优化：

- 生产预检不再调用 `auto_fit_reveal_boxes.py`。
- `prepare_full_slide_scenes.py` 改为精确 Mask 构建器的兼容别名，不再委托旧分层算法。
- README、AGENTS 和 QA 清单改为当前实际流程。
- 回归测试同时检查服务器和预检脚本不能引用旧生产算法。

保留但仅作诊断：

- `auto_fit_reveal_boxes.py`
- `split_master_layers.py`
- `decompose_slide_layers.py`
- `compose_manifest_layers.py`

## 验证策略

- Python AST/编译检查。
- JavaScript 语法和前端质量检查。
- Remotion TypeScript 检查。
- Remotion 生产依赖安全审计；升级受影响的间接依赖后，`npm audit --omit=dev` 为 0 项漏洞。
- 六步可见流程检查。
- Mask 完整性和旧算法隔离检查。
- 图片替换失效检查。
- 音频确认和尾部缓冲检查。
- 真实运行项目 Reveal、运行资产和浏览器视觉验证。
